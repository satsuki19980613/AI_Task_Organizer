# 習慣タブ ゲーミフィケーション化（気持ち族 育成システム）

## Context

ユーザーは継続が苦手で、習慣タブを「ゲーム感覚で続けられる仕組み」にしたい。
**1習慣＝1キャラ**として、たまごっち風のドット絵キャラがstreakに応じて成長していく育成システムを導入する。
キャラを通じて「自分のため」を「この子のため」に翻訳することで、継続のモチベーションを上げる。

### 確定済みの設計方針（ブレストで合意）

- **キャラ系統**：オリジナルテーマ「気持ち族」5系統 — **勇気 / 集中 / 安らぎ / 好奇心 / 喜び**
- **進化トリガー**：連続日数（streak）ベース ← 既存 `calculate_habit_stats()` の `currentStreak` を流用
- **退化あり、死亡なし**：streakが切れたらステージダウン。卵まで戻ることはあるがキャラは消えない
- **図鑑あり**：これまで育てたキャラを記録（最高到達ステージを系統別に保存）
- **ビジュアル**：ドット絵 16×16 を SVG として動的生成。スプライトシート画像は使わず、文字列配列で定義した各セルを `<rect>` で描画する方式に決定（プロトタイプで方針確定）
- **系統選択**：習慣作成時にユーザーが選ぶ（チャット経由・手動作成の両方）

## 進化・退化ルール

### ステージ（5段階）

| Stage | 名称 | streak閾値 | 状態 |
|---|---|---|---|
| 0 | たまご | 0–2日 | 初期状態 |
| 1 | ひな | 3日 | はじめての進化（早めの成功体験） |
| 2 | こども | 7日 | 1週間の壁を越えた証 |
| 3 | せいねん | 21日 | 「習慣化」と言われる目安 |
| 4 | せいたい | 60日 | 完全体 |

**退化ルール**：streak が 0 にリセットされた時点でステージを **1段階ダウン**（最低は0=たまご）。
これは「最後にdoneした日からの経過日数 > frequency許容」で判定する純粋関数として実装可能。

**現在ステージの導出**（推奨：純粋関数で都度計算、永続化しない）：
- 入力：`habit_logs` の全レコード
- 出力：`stage` (0–4), `streak`, `bestStage`（過去最高到達ステージ：図鑑用）
- ロジック：履歴を走査し、streakが伸びるたびに閾値超えで stage++、streakが切れるたびに stage-- (≥0) を再現する累積処理。
  → 純粋関数なので日々の log 状況を反映してリアルタイムに更新される。

ただし `bestStage` は履歴から再計算可能ながらコストがあるため、`habits` テーブルにキャッシュ列として持つのが現実的。

## 生活モーション（プロトタイプで設計済み）

キャラは部屋（ハビタット）の中で7種類の行動を自動でランダムに繰り返す。各行動は2フレームのドット絵を `setInterval` で切り替えて表現。

| 行動 | 動き | プロップ | フレーム速度 |
|---|---|---|---|
| ぼーっとする | まばたき・呼吸 | なし | 600ms |
| お散歩 | 歩行（足が交互） | なし | 220ms |
| お茶を飲む | 湯のみを胸→口元へ | 湯気付き湯のみ | 600ms |
| ごはん | 両手→口に運ぶ | おにぎり | 400ms |
| おひるね | 寝姿に切替 | Z's が浮かぶ | 1100ms |
| 考えごと | 首をかしげる | ？吹き出し | 700ms |
| よろこぶ | 着地↔ジャンプ | キラキラ | 200ms |

ステージ別に「できる行動」を制限：
- たまご：idle / think / happy のみ（動けない）
- ひな：idle / walk / happy / think / sleep
- こども以降：全行動

進化時は自動で `happy` 状態になり「うれしくて進化した！」と表示。
退化時は `think` 状態で「すこし元気がない…」と表示。

- `best_stage INTEGER NOT NULL DEFAULT 0` — 図鑑用、過去最高到達ステージ

既存パターン踏襲：[db.py:58](../db.py) の `ALTER TABLE habits ADD COLUMN color` と同じマイグレーション形式で追加。

新関数 `calculate_character_state(habit_id, today)` を追加：
- [db.py:258 calculate_habit_stats](../db.py) と同じ場所に実装
- streak と bestStage をDBから取り、現在 stage を導出
- 戻り値：`{stage, stage_name, streak, best_stage, species}`

### 2. API ([AI_Task_Organizer/app.py](../app.py))

- `GET /api/habits` レスポンスに `character` フィールドを追加
  ```json
  {"id":..., "title":..., "character": {"species":"courage", "stage":2, "stage_name":"こども", "streak":9, "best_stage":3}}
  ```
- `POST /api/habits` で `character_species` を受け取れるように
- `PUT /api/habits/<id>` で系統変更も可能に
- `POST /api/habits/<id>/log` の応答内で `bestStage` 更新処理を行う（stageが上がったら best_stage = max(best_stage, stage)）
- 新規 `GET /api/habits/zukan` — 図鑑用に系統別の最高到達ステージ集計を返す

### 3. キャラ表示UI ([AI_Task_Organizer/static/js/main.js](../static/js/main.js))

- 既存 `renderHabitView()` の各 `.hg-row` 左端にキャラ画像を追加
- スプライトデータと描画関数はプロトタイプ [docs/prototypes/sprite_preview.html](prototypes/sprite_preview.html) からそのまま移植可能
- `setInterval` でフレーム切替＋アクション切替の自動スケジューラを実装
- streak進化時に `happy` 状態へ即遷移＋進化エフェクト
- 退化検知時は `think` 状態＋退化エフェクト

### 4. 系統選択UI

- **手動作成モーダル**（既存があれば改修、なければ追加）：5系統のドット絵プレビュー＋名称＋意味の説明
- **チャット提案 `buildHabitProposalHtml`**：習慣作成確認時に系統選択UIを差し込む。AI側で習慣内容から推奨系統を `[[HABIT:...]]` 内で提示し、ユーザーは確定時に変更可

### 5. 図鑑タブ

習慣タブ内のサブビュー（既存のアナリティクス枠の隣 or 下）に「図鑑」セクションを追加：
- 5系統 × 5ステージのマトリクスで、到達済みは色付きドット絵、未到達はシルエット
- 各セルクリックで「いつ・どの習慣で到達したか」をモーダル表示

### 6. スプライトデータ

**方針変更**：当初は PNG のスプライトシートを `static/sprites/` に置く想定だったが、プロトタイプ検討の結果、**ドット絵を SVG として JS 内に文字列配列で持つ方式に変更**。

利点：
- 画像アセット不要（リポジトリが軽い）
- デザイン修正は1文字書き換えだけ
- 色テーマも PALETTE 定数で一括変更可能（同じ形のキャラを系統違いの色で使い回せる）
- フレームの派生が `frame(base, [[row,col,char],...])` ヘルパーで簡潔に書ける

データ規模の見積：
- 1キャラ × 5ステージ × 平均5アクション × 2フレーム = **約50フレーム/系統**
- 5系統で **約250フレーム**
- 1フレームは16×16の文字列配列（1KB弱）→ 全体で 250KB 程度、JS bundle としては許容範囲

### 7. CSS ([AI_Task_Organizer/static/css/style.css](../static/css/style.css))

- `.char-sprite` クラス：64×64 表示、`shape-rendering: crispEdges` でピクセル感維持
- 進化時アニメ `@keyframes char-evolve`、退化時アニメ `@keyframes char-devolve`
- アクション別のキャラCSSアニメ（c-idle / c-walk / c-tea / c-sleep / c-happy / c-think）
- プロップオーバーレイのアニメ（zzz-float / bubble-pop / sparkle）

詳細はプロトタイプ [docs/prototypes/sprite_preview.html](prototypes/sprite_preview.html) の `<style>` 部分をそのまま流用可能。

## 検証方法

1. **DBマイグレーション**：開発環境のDBで `python -c "from db import init_db; init_db()"` を実行し、既存習慣の `character_species` がデフォルト値で埋まることを確認
2. **手動テスト**：
   - 新規習慣を5系統それぞれで作成し、たまご状態で表示されることを確認
   - 開発用に「過去日付でlogを埋める」スクリプトで streak 3/7/21/60 を再現し、各ステージへの進化が反映されることを確認
   - streak を切らせる（数日サボる）と退化ステージで表示されることを確認
   - 図鑑タブで到達済み/未到達セルが正しく表示されることを確認
3. **ブラウザ動作確認**：preview MCP で実際にチェックボックスON→OFFを操作し、進化アニメ・退化アニメが動くこと、heatmap/streakバッジと整合することを確認
4. **既存機能の非破壊**：既存の習慣表示・heatmap・streakバッジ・チャット提案フローが全て従来通り動作することを確認
